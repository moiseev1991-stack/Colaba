# Coordination Log — координация между разработчиками и ИИ-агентами

**Цель:** не дублировать фичи, не конфликтовать по git, не расходиться в документации.

> Этот лог — **единственный общий контракт** между всеми участниками: разработчиками и ИИ-агентами (ZCode, Cursor, Claude Code и др.).
> Он агент-нейтрален. В полях «Кто» используйте имя разработчика, ник или идентификатор агента/IDE (напр. `Dev A`, `Dev B`, `cursor`, `claude`, `zcode`).

---

## 📋 Как работает

1. **Перед любой фичей** — прочитать этот файл (занята ли задача? свежая ли докука?)
2. **После любой фичи** — обновить `COORDINATION.md` + доки, которые трогали
3. **Синхронизация** — `commit → @user push → партнёр pull`
4. **Нашёл устаревшую доку** — обнови или отметь в блокере

## ✍️ Правила работы с документацией

### Когда обновлять доку

- **Всегда** после завершения фичи — если она меняет поведение, API, модель БД
- **Сразу** если нашёл расхождение между докой и реальностью
- **Перед push'ем** — проверить что `STATUS.md` и `ROADMAP.md` не отстали

### Источники правды (когда доки противоречат друг другу)

1. `docs/STATUS.md` — что реально работает, миграции, проблемы
2. `docs/ROADMAP.md` — куда движемся
3. `docs/colaba-brief-2026-05-29.md` — архитектура, стратегия, контекст
4. `CHANGELOG.md` — когда что вышло
5. Всё остальное — справочное, может устареть

### Кто отвечает

- **STATUS.md, ROADMAP.md, brief, карта docs/** — отвечающий назначается в таблице ниже (по умолчанию любой участник может обновлять)
- **Общее** — если трогаешь фичу → обнови её доку. Если не знаешь чья — отметь в блокерах

## ⚙️ Соглашения

### Git

- Ветки: `feature/<name>` (или `fix/<name>`, `docs/<name>`, `refactor/<name>`).
  При желании можно добавлять префикс участника: `feature/dev-a-<описание>`.
- PR только после `git rebase main`
- Conventional Commits: `feat:`, `fix:`, `chore:`, `docs:`, `test:`, `perf:` (проверяется commitlint)
- См. `AGENTS.md` §4 для полного git-workflow

### Миграции Alembic

- Перед созданием миграции — проверить что коллега не создаёт свою
- Номер миграции: следующий свободный (сейчас 032, следующая — 033)
- Если конфликт revision IDs — `alembic merge` или переименовать

### Координация

- Изменяешь COORDINATION.md → коммитишь → @user пушит
- Если задача занята — не начинать параллельную
- Спорные моменты решает @user
- Нашёл устаревший doc-файл → отметь в блокерах или обнови сам
