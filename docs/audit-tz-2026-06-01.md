Скопируй блок ниже целиком в Claude in Chrome.

## Контекст
Colaba — B2B leadgen-инструмент. Раздел «По картам» — поиск компаний в
2GIS с фильтрацией по содержимому отзывов, AI-оценка через LLM, контакты
с сайтов и со страниц 2GIS, карта Leaflet.

Со последнего аудита прокатились новые фичи (коммиты 7922482..7ac56fc):

1. **Side-drawer карточки**: открывается справа, ниже глобального
   навбара (top-14), карта/список слева остаются видны.
2. **Escape закрывает любую модалку**, крестик × имеет расширенный
   hit-area (40×40 фактически).
3. **Тёмная тема** прокатана по всему maps-разделу: карточки,
   фильтр-панель, drawer, страница пресетов, форма поиска, бейджи
   pain-тегов и AI-анализа — всё имеет dark-варианты.
4. **Контакты прямо в превью карточки списка**: phone/website/email +
   справа кнопка `[↗ 2GIS]` (deeplink на оригинальную карточку).
5. **HTML-парсер 2gis.ru/firm/{id}** работает на проде — Celery-таск
   `enrich_company_from_2gis_html` автоматически дёргается для каждой
   2gis-компании без телефона, парсит `__INITIAL_STATE__` JSON и regex
   по HTML, мерджит контакты в БД.
6. **Краулер сайта расширен**: ходит не только на homepage, но и на
   `/contacts`, `/contact`, `/kontakty`, `/about`, `/o-nas`. Извлекает
   Instagram/Facebook/OK/YouTube помимо Telegram/VK/WhatsApp.
7. **Встроенный пресет «Готовы к автоматизации»** + у 4 из 5 встроенных
   пресетов есть рекомендованный AI-промпт. Кнопка «Сохранить как мой
   пресет с AI» копирует промпт в user-пресет одним кликом.
8. **NICHE_PRESETS = 33** ниши (было 10).
9. **Карта Leaflet** с цветными маркерами + переключатель «Список / Карта».
10. **Страница `/app/leads/presets`** с кнопкой «Создать пресет», CRUD.
11. **Pain-теги больше не привязываются к positive отзывам** — позитивные
    цитаты не должны висеть под красными «негатив N».

## DESKTOP (1440×900) — светлая тема

### A. Side-drawer ниже навбара
1. Открой `/app/leads → По картам`, сделай поиск «стоматология / Москва»
   (или возьми из «Новый поиск» → быстрый старт).
2. Кликни любую карточку — drawer открывается **справа** (slide-in),
   занимая ~max-w-xl (≈576px). Карта/список **слева остаются видны**.
3. **КРИТИЧНО**: первый видимый элемент в drawer должен быть **название
   компании + крестик ×** в правом верхнем углу drawer. Они НЕ должны
   быть скрыты под глобальным навбаром. Drawer стартует с `top-14`.
4. Нажми **Escape** — drawer должен закрыться.
5. Открой снова, кликни **× в правом верхнем углу** — закрывается. Тач-зона
   должна быть удобной (фактически 40×40px).
6. Открой ещё раз, кликни по затемнённому фону слева — закрывается.

### B. Контакты в превью карточки списка
1. В списке найди карточку компании. Под бейджами (отзывы/негатив/
   отвечает/сайт) должна быть строка контактов:
   - Если в БД есть phone/email/website → иконки + кликабельные ссылки.
   - **Справа** должна быть кнопка **`[↗ 2GIS]`** (или `Я.Карты` для
     yandex-источника) — deeplink на `https://2gis.ru/firm/{id}`.
2. Кликни `[↗ 2GIS]` — открывается новая вкладка, реальная карточка
   2GIS этой компании. **БАГ если**: вкладка не открывается, или ведёт
   на главную 2GIS вместо конкретной фирмы.

### C. Контакты в drawer
1. Открой карточку. Под адресом — блок «Контакты».
2. Если у компании есть телефон/email/мессенджеры/сайт — все видны с
   иконками, кликабельны (`tel:`, `mailto:`, https-ссылки на t.me/vk.com/wa.me).
3. **Последней строкой блока** — ссылка «открыть исходную карточку»
   с иконкой и переходом на `https://2gis.ru/firm/{id}`.
4. Если контактов нет — мягкая подсказка «Контактов от провайдера нет…»
   + кнопка `[↗ Открыть в 2GIS]`.

### D. HTML-парсер 2GIS (Целевая проверка)
1. Сделай **новый** поиск, например «доставка цветов / СПб».
2. Дождись завершения парсинга (1-2 минуты, статус «готово»).
3. **Подожди ещё 3-5 минут** — Celery-таск `enrich_company_from_2gis_html`
   разгребает компании в фоне (rate-limit 20/мин = ~50 компаний за 2.5 мин).
4. Открой несколько карточек (хотя бы 5-10) — у части должны появиться:
   - телефон в основной строке контактов (если в БД был null)
   - Telegram / WhatsApp / VK / Instagram в блоке «Контакты»
5. Сравни **превью карточки в списке** до и после — счётчик заполненных
   контактов должен подрасти.

### E. Тёмная тема — главный тест
1. Включи **тёмную тему** через переключатель в правом верхнем углу
   (солнце/луна-иконка в шапке).
2. Перейди в `/app/leads → По картам`. Проверь визуально:
   - **Карточки в списке** — тёмный фон (slate-900), светлый текст.
     **Не должно** быть белых блоков на тёмном фоне.
   - **Бейджи** «негатив N / не отвечает / нет сайта / отвечает владелец»
     — должны быть **контрастные** (светлые буквы на полупрозрачном
     цветном фоне). НЕ светлые на светлом.
   - **Pain-теги** «Качество еды и сервиса» — янтарный полупрозрачный фон,
     светлая надпись.
   - **Боковая фильтр-панель**: фон тёмный, labels («Рейтинг», «Минимум
     отзывов»…) светло-серые, не зелёные/непонятные.
   - **Готовые пресеты** (кнопки слева) — тёмные с подсветкой при hover.
   - **Шапка выдачи** — заголовок белый, описание серое.
   - **Переключатель Список/Карта** — активный белый текст на светлом
     фоне (`slate-100` в dark mode инвертируется).
3. Открой drawer карточки — тоже должен быть тёмный, contacts/метрики/
   отзывы читаемые.
4. Открой `/app/leads/presets` — кнопки/таблица/модалки в тёмном:
   - Кнопка «Создать пресет» — светлая на тёмном фоне (инверсия).
   - Кнопки действий ✏👁🗑 — фон тёмный, иконки светлые.
   - Открой ✏ Редактировать — модалка тёмная, поля input/textarea тёмные.
5. Открой форму нового поиска (`Новый поиск`) — поля/чипы/кнопки в dark.
6. **БАГ если**: где-то ярко-белый блок «выпрыгивает» на тёмном фоне,
   или серый текст нечитаем на сером, или бейджи невидимы.

### F. AI-пресеты — авто-триггер
1. Открой `/app/leads/presets`. Если нет AI-пресета — нажми «Создать»,
   задай «Тест-авто-AI», AI-промпт «Оцени 0-10 готовность купить SMM.
   Ответ: число + одна строка», без фильтров, сохрани.
2. Вернись в «По картам», на форме поиска кликни этот пресет → зелёная
   плашка «Применён» должна содержать пометку **«AI-анализ запустится
   автоматически»**.
3. Запусти поиск. На странице результатов сразу появляется фиолетовая
   плашка AI с hint-текстом:
   - Пока парсер не закончил: «AI-анализ запустится автоматически, как
     только парсер завершит сбор компаний».
   - После terminal: «AI-анализ запустится автоматически. Можешь нажать
     кнопку чтобы запустить вручную».
4. Когда парсер завершится — анализ должен стартовать **сам**, без
   нажатия кнопки. Через 30-60 сек в карточках появятся бейджи `AI: N/10`.

### G. Сортировка по AI score
1. При активном AI-пресете в Select «Сортировка» доступны опции
   **«AI score ↓ (готовые сверху)»** и **«AI score ↑ (низкие сверху)»**.
2. Под Select видна подсказка «Компании без AI-оценки — в конце списка».
3. Переключи на «AI score ↓» — сверху уезжают компании с высокими
   оценками. Компании без оценки — в конце.
4. Сбрось AI-пресет (выбери другой пресет / новый поиск без AI) — опции
   AI score **исчезают** из Select.

### H. Карта Leaflet
1. В шапке выдачи — переключатель «Список / Карта». Кликни «Карта».
2. OSM-карта с цветными маркерами:
   - ≥4.5 или AI ≥7 — зелёный
   - 3.5-4.4 или AI 4-6 — жёлтый
   - <3.5 — красный, AI <4 — фиолетовый
   - без рейтинга/AI — серый
3. Цифра внутри маркера — рейтинг или AI score.
4. Кликни маркер → попап с именем, адресом, метриками, AI score,
   кнопкой «Открыть карточку».
5. Кликни «Открыть карточку» → drawer открывается **справа**, карта
   слева остаётся видна.
6. Закрой drawer — карта на том же зуме/позиции.

### I. «Сохранить как мой пресет с AI»
1. На форме поиска нажми «Готовы к автоматизации» (новый встроенный
   пресет) или «Кризис репутации».
2. Должна появиться фиолетовая плашка «У этого пресета есть готовый
   AI-промпт — сохрани как свой, чтобы запустить анализ» + кнопка
   **«Сохранить как мой пресет с AI»**.
3. Кликни — открывается модалка с **пред-заполненными** полями:
   - name = «Готовы к автоматизации + AI»
   - AI-промпт = осмысленный текст про CRM/RPA/чат-боты
4. Сохрани → пресет применяется автоматически (видишь в зелёной плашке).

### J. Pain-теги — позитивные цитаты больше не висят
1. Запусти поиск ниши с большим числом отзывов («стоматология Москва»).
2. Подожди 3-5 минут пока AI-анализ отзывов отработает (бейджи pain
   появятся в карточках).
3. Под красным/жёлтым pain-тегом («Качество», «Хаос в работе») цитата
   должна быть **негативной или нейтральной**, не «Очень вкусная кухня,
   вежливый персонал».
4. **БАГ если**: видишь позитивную цитату под pain-тегом — значит
   SQL-чистка не прошла или фикс не работает.

### K. Расширенный краулер сайта
1. Найди компанию с **сайтом** (бейдж «есть сайт» в карточке).
2. Подожди 2-3 минуты после первого парсинга (`enrich_company_contacts`
   таск должен отработать).
3. Открой drawer этой компании — в блоке «Контакты» должны быть не
   только email/телефон с главной, но и Telegram / VK / WhatsApp /
   Instagram / Facebook / OK / YouTube если они есть на сайте.

### L. Регрессии
- Сортировки «Рейтинг ↓/↑», «Больше отзывов», «Больше негатива»,
  «По упоминаниям болей» работают.
- Применение пресета «Нужен сайт» (без AI) — работает.
- Экспорт CSV — скачивается.
- Зелёный баннер «Применён пресет с формы поиска» **исчезает** как
  только юзер крутит фильтр в боковой панели.

## MOBILE (390×844, iPhone 14 в DevTools)

DevTools (F12) → Toggle device toolbar (Ctrl+Shift+M) → iPhone 14
(390×844) → перезагрузи страницу.

### M1. Форма поиска
- Чипы пресетов фильтров и популярных ниш — **в один ряд с горизонтальным
  скроллом** (не разваливаются в 4 строки).
- Поля «ниша», «город» удобно тапать.

### M2. Карточки в выдаче
- Кнопки «В список» / «Письмо» — **min-height 36px** (палец попадает).
- Кнопка `[↗ 2GIS]` в правой части строки контактов — кликабельна.

### M3. Drawer на mobile
- Открой карточку — drawer занимает **всю ширину** экрана.
- **КРИТИЧНО**: крестик × видимый и кликабельный (не уехал под навбар).
- На iPhone с нотчем (iPhone 14 Pro 393×852) контент не обрезается
  снизу — `safe-area-inset-bottom`.
- Escape с клавиатуры закрывает.

### M4. Карта
- Влезает по ширине, пинч-зум работает.
- Маркеры читаемы.

### M5. /app/leads/presets на mobile
- Кнопки действий ✏ 👁 🗑 — **min-h-9 min-w-9** (36×36 минимум).
- Кнопка «Создать пресет» в шапке видна, кликабельна.

### M6. Тёмная тема на mobile
- Включи dark, открой `/app/leads → По картам` на mobile-viewport.
- Карточки тёмные, бейджи контрастные, нет белых выпрыгивающих блоков.

### M7. Known (не фиксировать)
- Боковая фильтр-панель на странице результатов уезжает над списком на
  mobile (не bottom-sheet) — отмечено как известная задача.

- НЕ тестируй ЛК, email-кампании, лиды-по-сайтам — только «По картам».
- НЕ фикси сам, только описывай.

  пунктом и остальное пропусти. Должно быть видно коммит `7ac56fc` или
  свежее.
