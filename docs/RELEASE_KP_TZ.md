> Источник: запрос юзера 2026-06-12. Документ зафиксирован, чтобы

## 0. Контекст и цель

**Целевая аудитория продукта (зафиксирована):** те, кто продаёт услуги малому
бизнесу — SEO-фрилансеры, веб-студии, маркетинговые агентства, AI-энтузиасты
с сервисами для бизнеса.

**Главный сценарий (единственный, всё остальное вторично):**

```
ниша + город → выдача компаний с болями → фильтр → кнопка «КП» →
драфт письма под конкретную боль с цитатой из отзыва → копировать/отправить
```

**Критерий готовности релиза:** новый пользователь из рекламы за 10 минут
самостоятельно проходит путь регистрация → первый сгенерированный КП.

**Не входит в этот релиз (НЕ ТРОГАТЬ, даже если кажется логичным):**
тендеры, hh.ru, white-label, публичный API, гео-отчёты для франшиз,
реверс-ORM для SMB, Google-провайдер карт, развитие heatmap.

## 1. Эпик A — Кнопка «КП» как центр продукта

### A1. КП из выдачи (frontend)
- В `MapsSearchResults.tsx` в каждую строку компании добавить кнопку `КП`
  (иконка + текст, primary). Клик открывает `DraftEmailModal` без drawer.
- В `MapsCompanyDetailDrawer.tsx` поднять блок КП на первый экран drawer.
- Кнопка disabled с tooltip «У компании нет проанализированных отзывов»
  если pain_tags пусты и AI-анализ не завершён.

### A2. Переработка генерации КП (backend)

```
POST /outreach/kp/generate
body: { company_id: int, template_key: str, tone: "neutral"|"bold" = "neutral" }
response: {
  id, subject, body,
  arguments_used: { pain_label, quote, trend, source },
  remaining_free: int | null
}
```

Контекст промпта:
1. Компания: name, niche, city, website (есть/нет), rating.
2. Топ-боль: `CompanyPainScore` max `mention_count` → label + `top_quote`.
3. Тренд: rising/falling/stable + период.
4. Бенчмарк: упоминается ли боль чаще, чем в среднем по нише.
5. Профиль отправителя из `template_key`.

### A3. Шаблоны КП

Таблица `kp_templates`:
`id, key (unique), title, sender_profile, offer_hint, is_system, organization_id (nullable), created_at`

4 системных шаблона:

| key | title | sender_profile | offer_hint |
|---|---|---|---|
| `webstudio` | Веб-студия / разработка | «веб-студия, делаем сайты, онлайн-запись, интернет-магазины» | нет сайта / жалобы на запись и дозвон → предлагаем сайт или модуль записи |
| `seo` | SEO / продвижение | «специалист по продвижению в поиске и на картах» | рейтинг ниже среднего по нише / мало отзывов → предлагаем работу с репутацией и видимостью |
| `marketing` | Маркетинг / реклама | «маркетинговое агентство, приводим клиентов» | растущий негатив / отток клиентов → предлагаем удержание и привлечение |
| `custom` | Свой вариант | пользователь вводит 1–2 предложения о себе в модалке | — |

Эндпоинт: `GET /outreach/kp/templates`.

### A4. Промпт генерации (константа в коде)

```
Ты пишешь короткое холодное коммерческое предложение на русском языке
от лица: {sender_profile}.

Получатель: {company_name} — {niche}, {city}.
Факты о получателе (используй ТОЛЬКО их, ничего не выдумывай):
- Главная боль клиентов по отзывам: «{pain_label}» ({mention_count} упоминаний)
- Цитата из реального отзыва: «{top_quote}»
- Динамика: {trend_phrase}
- Сравнение с конкурентами: {benchmark_phrase}
- Сайт: {website_status}
- Рейтинг: {rating} (средний по нише {niche_avg})

Подсказка по офферу: {offer_hint}

Требования:
1. Тема: до 8 слов, без кликбейта, с конкретикой про их бизнес.
2. Объём: 110–160 слов. Абзацы по 1–3 предложения.
3. Структура: (а) наблюдение по их отзывам с цитатой, (б) что это им стоит,
   (в) что конкретно предлагаем, (г) лёгкий CTA — короткий созвон или ответ.
4. Тон: {tone}. Уважительный, на «вы», без лести и канцелярита.
5. ЗАПРЕЩЕНО: «уникальное предложение», «динамично развивающаяся компания»,
   «инновационный», обещания гарантий, выдуманные цифры, упоминание ИИ
   и того, как мы их нашли.
6. Не вставляй плейсхолдеры вида [Имя].

Верни строго JSON: {"subject": "...", "body": "..."}
```

- Парсинг с ретраем (1) на битый JSON, потом 422.
- Сохранение в `kp_drafts`: id, user_id, organization_id, company_id,
  template_key, subject, body, arguments_used jsonb, created_at.

### A5. Модалка КП (фронт, переработка DraftEmailModal)
- Селект шаблона + поле «о себе» для `custom`.
- Кнопки: «Сгенерировать заново», «Копировать письмо», «Копировать тему».
- Блок «Аргументы» под письмом: боль, цитата, тренд.
- Счётчик «Осталось N бесплатных КП», на 0 — пейволл (Эпик E).

### Acceptance criteria (A)
- [ ] КП в 1 клик из строки выдачи и из drawer.
- [ ] Письмо с цитатой из отзыва, без клише, 110–160 слов.
- [ ] Смена шаблона меняет оффер, перегенерация даёт другой текст.
- [ ] Все драфты в `kp_drafts`.
- [ ] pytest на сборку контекста + парсинг LLM.

## 2. Эпик B — Онбординг «3 шага до КП»

1. «Кто вы?» — 4 карточки (Веб-студия/SEO/Маркетинг/Другое) → template_key + пресет.
2. «Где ищем клиентов?» — ниша + город + 6 примеров-чипов.
3. «Поехали» — поиск, прогресс, редирект в выдачу.

Demo-режим: `DEMO_SEARCH_ID` в `.env`, read-only выдача из кэша.

Метрика: `onboarding_events` или существующая аналитика.

## 3. Эпик C — Пресеты под профессии

- `is_system` boolean в user_filter_presets.
- 3 системных пресета:

| key | название | фильтры |
|---|---|---|
| `for_webstudio` | Для веб-студий | нет сайта ИЛИ боль из кластеров записи/дозвона; рейтинг ≥ 3.5 |
| `for_seo` | Для SEO | рейтинг ниже среднего ИЛИ отзывов < медианы ниши; сайт есть |
| `for_marketing` | Для маркетологов | тренд негатива = rising; отзывов ≥ 10 |

«боль из кластеров записи/дозвона» — словарь подстрок (`запис`, `дозвон`,
`телефон`, `ждать`) в конфиге.

UI: ряд chips над выдачей; активный подсвечен.

## 4. Эпик D — Динамика и вхождения как аргумент в КП

- `trend_phrase` и `benchmark_phrase` собираются детерминированно в Python
  (не LLM) из существующих эндпоинтов negative-trend, pain-benchmark.
- Если stable/мало данных — фразы опускать.
- В строке выдачи рядом с топ-болью — мини-бейдж тренда (если rising).

## 5. Эпик E — Касса (ЮKassa) и free-лимиты

Тарифы:

| plan | цена | поисков/мес | КП/мес |
|---|---|---|---|
| `free` | 0 | 1 | 5 |
| `start` | 2 900 ₽/мес | 10 | 100 |
| `pro` | 7 900 ₽/мес | 50 | 500 |

Enforcement — dependency на `POST /maps/search` и `POST /outreach/kp/generate`,
402 с `{error: "limit_reached", limit_type, plan, upgrade_url}`.

ЮKassa: redirect-флоу, webhook идемпотентен, модель `payments`. Без
автосписаний на старте.

`/billing`: текущий план, расход, 2 кнопки апгрейда. Пейволл-модалка с
конкретным текстом под тип лимита.

E3: `spinlid.io` → `spinlid.ru` (на 2026-06-12: уже 0 вхождений в репо).

## 6. Эпик F — «Поиск по сайтам»

Вкладка «Карты» | «Сайты» в общем layout. Форма + 4 пресета-примера:
- «© 2021» — заброшенные сайты
- «Joomla» / старая CMS — миграция
- «доставка по телефону» — нет интернет-магазина
- «страница в разработке» — недоделанные

Результаты + кнопка «КП». Контекст промпта для site-КП — отдельная ветка
с {url} и {entry} + {entry_meaning} из словаря.

Модель `SiteLead`: id, user_id, query, url, domain, snippet, entry, created_at.
В `kp_drafts` добавить `site_lead_id` nullable, `company_id` сделать nullable
+ CHECK.
