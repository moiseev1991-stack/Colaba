## 0. Главный итог

**Продукт живой и основной flow работает** — UI поиска лидов (главная фича) проходит end-to-end (форма → backend → Celery → результаты + 30+ фильтров + AI-кластеры болей). Backend стабилен (нет 500), миграции согласованы с моделями (head=042), авт-сессия работает на большинстве страниц.

**Но накопилось критичного:**
1. 🔴 **Был каскадный 500 на 100% UI** из-за отсутствия `leaflet` в `node_modules` volume (исправлено в процессе аудита).
2. 🔴 **5 endpoints без auth в `/maps/insights/*` и `/maps/pain-tags`** — утечка бизнес-know-how (ниши, боли, AI-кластеры) кому угодно.
3. 🔴 **`/maps/admin/*` без `require_superuser`** — любой залогиненный юзер может запустить тяжёлые Celery-задачи (DoS/абуз).
4. 🔴 **Front обращается к несуществующему `/outreach/templates`** — все «сохранённые шаблоны» пользователя живут только в localStorage.
5. 🟡 **4 страницы email-модуля полностью пустые** (`/app/email/replies`, `/stats`, `/settings`) + **`/profile` теряет сессию** + **`/app/leads/kp-jobs` 404** + **история поиска не обновляется**.

## 7. Cost tracking MVP (2026-07-05, ветка `feature/cost-tracking-api-log`)

Новая фича (не закрытие дыр из этого аудита, а развитие) — система учёта
внешних API-вызовов в таблице `api_call_log` (миграция 044). Раньше
модуль `monitor` был mock-ером с захардкоженными данными (§1.4): теперь
он читает реальные вызовы и считает стоимость в рублях.

### Что сделано

- **Модель `ApiCallLog`** (16 полей): provider, endpoint, http_status,
  latency_ms, ok, prompt_tokens/completion_tokens, model, cost_rub,
  контекст (user_id, map_search_id, company_id).
- **`provider_pricing.py`** — статичная таблица цен в ₽ для 11 провайдеров:
  2gis, serpapi, dadata, 2captcha, anticaptcha, openai, anthropic,
  openai_emb, hyvor, smtp, yookassa. Функция `compute_cost_rub()`.
- **Трекер `api_tracker.log_call()`** — fire-and-forget запись, не падает
  при ошибках БД. Контекст через contextvars (current_user_id/
  map_search_id/company_id), выставляется в Celery-tasks и FastAPI
  middleware.
- **Интегрированные точки вызова**: `_request()` в 2gis/serpapi/dadata,
  `_chat_openai`/`_chat_anthropic`/`_chat_openai_compatible`/ollama в
  `ai_assistants/client.py`, `embed_texts` в `reviews_ai/llm.py`,
  `_send_via_hyvor`/`_send_via_smtp` в `email/service.py`.
- **Monitor real data**: `/monitor/requests` отдаёт последние вызовы,
  `/monitor/summary?period=day|week|month|all` — агрегат с breakdown
  по провайдерам, `/monitor/by-search/{id}` — стоимость конкретного поиска.
- **Frontend**: `RequestMonitorTable` показывает Provider/Endpoint/
  Method/Time/Cost₽/OK (раньше было Method/URL/Response/Phone/OK с mock).

### Что осталось открытым

- **Captcha solver integration**: `2captcha`/`anticaptcha` есть в таблице
  цен, но точки вызова в `solver.py` не обёрнуты трекером. Добавить при
  необходимости (5-минутная правка).
- **Per-user квоты и энфорсмент**: счётчики есть, но проверки «не превысил
  ли план» нет. Полный биллинг — следующий этап.
- **TTL-чистка**: `api_call_log` растёт (~5K/день при 10 поисках); нужен
  cron/celery-beat таск на удаление старых записей.
- **Динамические цены**: только статика, обновляется вручную.

## 8. Email-провайдеры с fallback (2026-07-05, ветка `feature/email-providers-fallback`)

Новая фича — 3 канала отправки email с приоритетами и авто-переходом на
резервный при сбое. Раньше был бинарный выбор `email_config.provider_type ∈
{hyvor, smtp}` без fallback и без указания цены письма.

### Что сделано

- **Модель `EmailProviderConfig`** (миграция 045): singleton-per-provider
  (3 строки postbox/ses/hyvor). Поля: smtp_host/port/user/password,
  api_key, secret_key, from_email/name, region, **cost_per_mail** (₽ за
  письмо — админ задаёт в UI), is_enabled, is_configured, priority
  (0=primary, 1=fallback, 2=tertiary).
- **Реестр `providers_registry.py`** — статичные метаданные для UI (имя,
  описание, поля с типами/дефолтами).
- **Сервис `providers_service.py`** — CRUD с секрет-маской, `get_active_chain`
  для fallback, `set_priority` с перестановкой.
- **Роутер `providers_router.py`** (mount `/email/providers-settings`):
  GET (3 провайдера), PUT (save), PUT `/priority` (перестановка),
  POST `/test` (реальный SMTP-connect для postbox/ses, HTTP-ping для hyvor).
- **Fallback в `send_email`**: перебор включённых провайдеров по приоритету;
  при `EmailServiceError` переходит к следующему. Каждый успешный send
  логируется в `api_call_log` с `cost_rub = cost_per_mail` провайдера —
  это прямое решение вопроса «где задавать стоимость отправки».
- **Frontend**: страница `/app/settings/email-providers` — 3 карточки в
  стиле maps-providers с чекбоксом «Включён», кнопками ↑/↓ приоритета,
  полем «Цена за письмо (₽)», кнопками «Проверить»/«Сохранить».
- **Sidebar**: «Провайдеры email» в секции настроек всех модулей.

### Обратная совместимость

Старая таблица `email_config` НЕ удалена — она хранит IMAP/подпись/лого.
Если в новой таблице все 3 провайдера отключены, `send_email` откатывается
к старой логике `provider_type` (hyvor/smtp).

### SMTP-тест

`POST /email/providers-settings/{id}/test` для postbox/ses делает реальный
SMTP-connect: `aiosmtplib.connect(use_tls=..., start_tls=...)` + login.
Возвращает понятную ошибку если кредентиалы неверны
(`SMTPAuthenticationError: 538 Authentication credentials invalid`) — это
значит тест работает: когда пользователь введёт реальные ключи Postbox,
тест вернёт `ok:true`.

Полное описание — `docs/guides/EMAIL_PROVIDERS.md`.
