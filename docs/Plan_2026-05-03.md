# Plan — Colaba product strategy & architecture extensions

**Дата:** 2026-05-03
**Контекст:** solo-фаундер, без команды, без бюджета, РФ B2B рынок 2026 (падающий)

---

## Часть 1. Прямой ответ: продукт или нет

Сейчас Colaba — это **рабочий движок без продуктовой оболочки**.

Архитектурно очень плотный B2B-комбайн: поиск → краулинг → SEO-аудит → контакты → email-рассылка → ответы (IMAP) → AI-ассистенты → multi-tenant → OAuth → админка.

Но **продать это нельзя**, потому что нет ни одной из вещей, без которых клиент платит:
- self-service биллинга,
- onboarding,
- нишевой обёртки (продукт для всех = продукт ни для кого),
- доказательств ценности,
- узкого ICP.

Pricing-страница есть, но «Подключить Pro» → «по запросу» = ручные продажи = в одного не масштабируется.

---

## Часть 2. Что есть как актив (не недооценивать)

- Поисковый стек с фолбеками: DuckDuckGo (free) + Яндекс XML/HTML + Google HTML
- **Hyvor Relay** в стеке — своя SMTP-инфра, не зависим от Mailgun/SendGrid (важно для РФ)
- Парсинг ответов IMAP — мало кто из no-code outreach это умеет
- Capcha bypass: AI Vision + 2captcha + anticaptcha + Yandex SmartCaptcha
- Multi-tenancy + RBAC + OAuth (Google/Yandex/VK/Telegram готовы, нет credentials)
- Celery-конвейер на 6 воркеров — масштабируется
- Модуль **тендеров (госзакупки)** заложен — для РФ это огромная самостоятельная вертикаль

Инженерный долг до MVP небольшой. Долг продуктовый и маркетинговый — большой.

---

## Часть 3. Критические дыры до состояния «можно платить»

### Блокеры (без них не продать)

1. **Self-service биллинг.** ЮKassa или CloudPayments + рекуррентные платежи + лимиты по тарифам в коде. Сейчас pricing — статичная HTML-секция, в БД лимиты не enforced.
2. **Onboarding за 60 секунд.** Регистрация → готовый «рецепт» поиска → первые 10 лидов в браузере без настройки. Сейчас новичок попадает в дашборд из 6 разделов и не понимает, с чего начать.
3. **Доменная инфра для cold email.** SPF/DKIM/DMARC + warm-up + ротация доменов + bounce/complaint обработка. Hyvor Relay есть, но нет UI для подключения 5 доменов «под отправителя», прогрева, парковки. Это причина №1, почему cold-email-сервисы не работают.
4. **Demo-аккаунт / песочница.** Любой клиент в РФ 2026 не зарегается, пока не увидит интерфейс изнутри.
5. **Реальные кейсы.** Минимум 2–3 скриншота с «было/стало»: 100 лидов за час, 12 ответов, 2 сделки.

### Важно, но можно после первых продаж

6. Sentry + rate limiting + SQLAdmin auth (security hygiene)
7. Telegram outreach — для РФ это **главный** канал, важнее email
8. Юр. часть: 152-ФЗ (персональные данные), оферта, согласие — без этого штрафы РКН
9. SEO-семантика самой Colaba — единственный бесплатный источник трафика для solo-фаундера

## Часть 4. Стратегия под РФ 2026 (падающий рынок, solo, без бюджета)

Падающий рынок означает две вещи: **(а) у клиентов нет денег на ads → они ищут лидов дешевле** и **(б) у вас нет денег на ads → ищите канал привлечения, не зависящий от бюджета.** Эти два факта работают на вас, если правильно спозиционироваться.

### Сужать ICP до одной ниши на квартал

«Лидогенератор для всех» = не продаётся. Один из вариантов:

- **SEO-агентства / частные SEO-специалисты** → им нужны клиенты с плохим SEO; ваш SEO-аудит + контакты = готовый КП в одну кнопку. ICP: 5–50 чел., оборот 1–10 млн ₽/мес. В РФ ~3–5 тыс. таких.
- **Тендерные специалисты** → ваш модуль `tenders` + автомониторинг новых закупок + outreach к заказчикам. Ниша платит $50–200/мес охотно.
- **Веб-студии** → им нужны заявки от компаний с устаревшими сайтами. Краулер находит сайты без SSL/мобайла/HTTPS = готовый список клиентов.

Рекомендация: **SEO-агентства** или **тендерщики** — у них уже есть привычка платить за инструменты.

### Канал привлечения для solo без бюджета

1. **Контент-SEO**: 30 статей «как найти клиентов на [ниша]» на blog.colaba.ru. Ваш ИИ-ассистент пишет тексты под Yandex. 6 месяцев работы → вечный канал.
2. **Eat your own dog food публично.** Telegram-канал: «На этой неделе наш бот нашёл 1247 заводов с битыми сайтами в Свердловской обл., написал им, получил 23 ответа». Demo + контент + lead-magnet одновременно.
3. **Telegram-чаты ниши**: 20–30 чатов SEO/маркетинга, ненавязчивые ответы → личка → demo.
4. **Партнёрка с агентствами**: 30% recurring за приведённого клиента.

### Что вырезать (для solo это критичнее, чем что добавить)

- **OAuth VK и Telegram Login Widget** — никто из B2B так не логинится. Оставить Email + Yandex.
- **AI-ассистентов как раздел** — это внутренний инструмент, спрятать в шестерёнки.
- **WhatsApp в roadmap** — для РФ B2B не нужен, плюс Meta-блок.
- **Госзакупки и SEO-аудит** — две разные продуктовые сущности. Выбрать одну на старт.

## Часть 5. План 30 / 60 / 90 дней

### 30 дней — превратить движок в покупаемый MVP
- [ ] Выбрать одну нишу (SEO-агентства / тендерщики / веб-студии)
- [ ] Удалить из UI всё, что не относится к нише
- [ ] Demo-аккаунт с предзагруженными данными
- [ ] ЮKassa интеграция + 2 тарифа (free 50 лидов/мес, paid 2900 ₽/мес 5000 лидов)
- [ ] Onboarding-tour из 3 шагов
- [ ] SQLAdmin закрыть авторизацией, Sentry подключить (free tier)

### 60 дней — найти первых платящих
- [ ] Telegram-канал с публичными «прогонами» (2 поста/нед.)
- [ ] 3 кейса на сайте (даже по знакомым)
- [ ] Партнёрская программа (UTM + промокоды)
- [ ] Email warm-up для Hyvor Relay (1 домен, 4 недели прогрева)
- [ ] Cold outreach самих себя: 200 агентств вашим же продуктом → demo-call

### 90 дней — чёткая unit-экономика
- [ ] CAC и LTV на 5–10 платящих
- [ ] Юридика: оферта, политика 152-ФЗ
- [ ] Решение: PMF есть → масштабируем; PMF нет → меняем ICP

**Реалистичный таймлайн до первых платящих:** 60–90 дней при 4–6 ч/день и фокусе. Без фокуса — 12 месяцев и выгорание.

## Часть 6. Расширения на базе текущей архитектуры

Все идеи ниже **переиспользуют существующую инфраструктуру** (Celery, captcha bypass, search-абстракция, AI-ассистенты, Hyvor Relay) — это и есть ваше конкурентное преимущество. Не делайте всё сразу: выберите 1–2 расширения после стабилизации MVP.

### A. Парсинг карт — 2GIS / Яндекс.Карты / Google Places ⭐ топ-приоритет

**Зачем:** карты — самый чистый источник B2B-лидов в РФ. Конкретный пример запроса клиента: «найди мне все автосервисы в Казани с рейтингом ≤4.0 и без сайта». Это превращает Colaba из абстрактного «лидогенератора» в **«найди мне 1000 потенциальных клиентов за 5 минут»**.

**Реализация (1–2 недели):**
- Новый провайдер `MapsProvider` по тому же интерфейсу, что текущий `SearchProvider`
- **2GIS** — самый полный по РФ; есть [официальный API](https://docs.2gis.com/) (есть free tier для разработчиков), плюс можно парсить публичный поиск
- **Яндекс.Карты** — Geocoder API + Places API (платный, но дешёвый); либо HTML-парсинг через ваш capcha bypass
- **Google Places** — $200/мес free credit, для РФ менее ценно
- Экстрактор: название / адрес / телефон / email (часто есть) / сайт / категория ОКВЭД / рейтинг / часы работы / соцсети
- Сразу пропускайте через ваш SEO-аудит → знаете «у кого сайт убит»
- Фильтры: регион, рейтинг, наличие сайта/SSL, категория, число отзывов

**Юр. риск:** парсинг публичных карт — серая зона. Безопаснее использовать **официальные API 2GIS/Яндекса** в платных тарифах. Стоит копейки на первых клиентах.

**Монетизация:** «Map Lead Pack» как отдельный SKU — 990 ₽ за 1000 организаций по фильтрам.

---

### B. Авто-заполнение форм заявок (form auto-submit)

**Зачем:** клиенты массово хотят: «у меня есть 500 сайтов с формой обратной связи — напиши им через форму, не email». Email RU доставляется плохо, а формы → 100% доходят.

**Что есть в архитектуре:** capcha bypass + краулер + AI Vision уже работают. Нужно надстроить:
- **FormDetector**: heuristic (поле email, поле message, submit) + GPT-4 Vision как fallback на скриншоте страницы
- **Playwright**-сценарий заполнения с подменой User-Agent + прокси-ротация
- Лог: попытка / успех / отказ / редирект на капчу
- Шаблонизатор сообщений (можно делиться с email-шаблонами)

**Юр. риск (важно!):** массовое заполнение чужих форм нарушает ToS многих сайтов и 152-ФЗ (ст. 18 — обработка ПД третьих лиц). **Позиционирование:** «отправка ваших собственных заявок от вашего лица» (т.е. контент явно от клиента, не спам). На фронте обязательно: согласие пользователя «вы заявляете, что ваше сообщение — добросовестное коммерческое обращение».

**Реализация:** 2–3 недели. Сильно увеличивает unit-экономику клиента (формы = в 5–10 раз выше отклик чем cold email РФ).

---

### C. Tender autobid / госзакупки — отдельный продукт на той же инфре ⭐

**Зачем:** ниша, где люди уже платят $50–200/мес (Synapsenet, Тендерплан, СБИС-Тендеры). Огромный TAM в РФ.

**Что есть:** модуль `tenders` уже заложен в архитектуре + Celery + AI-ассистенты.

**Что добавить:**
- Парсер [zakupki.gov.ru](https://zakupki.gov.ru/epz/main/public/document/view.html) — есть **открытый API** ЕИС (FZ-44 + FZ-223)
- **Фильтры**: ОКПД2, регион, цена контракта, ключевые слова, заказчик
- **Уведомления в Telegram** при появлении подходящего тендера (за 1 минуту до конкурентов)
- AI-ассистент: «оцени, подходит ли тендер моей компании по описанию» (RAG по профилю клиента)
- Авто-генерация **первичной заявки** (форма 1) — заполнение типовых полей; ручная допиливка заказчиком
- Парсинг тендеров **коммерческих** площадок: B2B-Center, Roseltorg, Сбер-А

**Монетизация:** 2900–4900 ₽/мес. Ниша готова платить, рынок зрелый.

**Это потенциально может быть отдельный спин-офф** — `Colaba Tenders` под отдельным доменом, тот же codebase.

---

### D. Reputation monitoring (отзывы)

**Зачем:** рестораны, клиники, автосервисы, ритейл — все хотят знать, что про них пишут.

**Что есть:** краулер + AI-ассистент.

**Что добавить:**
- Парсеры: Я.Карты, 2GIS, Otzovik, Flamp, Irecommend, Zoon
- Diff-механизм (новый отзыв → событие)
- AI-классификация тональности (positive / negative / neutral) + темы
- Дашборд: график динамики, топ жалоб, alerts
- Auto-reply: AI-ассистент готовит черновик ответа на негатив

**Монетизация:** 1990 ₽/мес за 1 организацию, 4990 ₽/мес за сеть до 10 точек.

---

### E. AI-генерация коммерческих предложений (КП)

**Зачем:** замыкает цикл: лид → персональное КП → отправка. Сейчас КП рисуют руками. У вас уже есть и SEO-аудит, и контакты, и AI.

**Что добавить:**
- Шаблоны КП (PDF/HTML) с подстановкой данных лида
- Генератор «персонального абзаца»: «у вас сайт грузится 8 секунд, мы можем ускорить до 1.5» (на базе SEO-аудита)
- Подпись + бренд клиента
- Трек открытия КП (pixel / уникальный url)

**Это улучшение существующего outreach, а не отдельный продукт.** Сильно поднимает Conversion от рассылки.

---

### F. Web monitoring / change detection

**Зачем:** конкурентный анализ, мониторинг цен, отслеживание новостей у клиентов/конкурентов.

**Что есть:** краулер.

**Что добавить:**
- Хранение снапшотов страниц
- Diff-движок (text + DOM-структура)
- Триггеры: «цена < X», «появилось слово Y», «изменился баннер»
- Алёрты в email/Telegram

**Монетизация:** $20–50/мес за 100 страниц мониторинга. Простой продукт, понятная ценность.

### G. Enrichment — обогащение данных лида

**Зачем:** одно дело найти `info@company.ru`, другое — знать выручку, ИНН, число сотрудников, ОКВЭД.

**Источники для РФ:**
- **List-org** (есть HTML-парсинг, бесплатно с лимитами)
- **ОФД** через [www.nalog.ru](https://www.nalog.ru/)
- **Контур.Фокус** — платный API (~5 ₽/запрос)
- **СПАРК** — платный, но точный

**Реализация:** 1 неделя. Ставится как «Enrich button» рядом с лидом в UI.

**Монетизация:** включено в Pro, как differentiator vs. конкурентов.

### H. AI Voice Sales Agent (звонки)

**Зачем:** в РФ 2026 это новая бурно растущая ниша. Минимум зрелых решений.

**Что добавить:**
- Интеграция с Yandex SpeechKit (TTS+STT в РФ работает; ElevenLabs тоже работает через прокси)
- GPT-4 Realtime / Yandex GPT для диалога
- VoIP-провайдер: Mango Office / SIPNet / Telphin
- Сценарии: квалификация лида (3–5 вопросов), назначение встречи

**Сложность:** высокая. Это потенциально **второй продукт** через 6–12 месяцев, не сейчас.

**Монетизация:** премиум, 9900+ ₽/мес.

**Что добавить:**

**Монетизация:** входит в основной тариф для SEO-ниши.

### J. CRM-light / Lead pipeline

**Зачем:** у вас уже есть лиды, ответы (IMAP), шаблоны. Это уже почти CRM.

**Что добавить:**

- Заметки + следующее действие

**Это удерживает клиента в продукте.** Без CRM он сольёт лиды в свой Битрикс и забудет про вас.

## Ключевые принципы выбора что делать дальше

1. **Не строить вширь, пока нет 5 платящих.** Каждое новое расширение разбавляет фокус и удваивает поддержку.
2. **Расширение должно использовать ≥80% существующего кода.** Если для фичи нужен новый сервис/сторадж/прокси-сетка — это не расширение, это второй продукт.
3. **Каждое расширение должно дать новый SKU или поднять ARPU.** Не делайте «улучшения для красоты» — клиенты их не замечают.
4. **Tender, карты, формы — это три самостоятельных продукта.** Не пытайтесь продать одному клиенту все три. Один продукт = одна landing = одна целевая страница в Я.Директе.

## Что делать прямо сейчас (на этой неделе)

1. Выбрать нишу из трёх: **SEO-агентства / тендерщики / веб-студии**
2. Решить: на 30 дней фокус — превращение в продукт (часть 5) или сразу делать карты (часть 6.A)
3. Если фокус = продукт: первая задача — ЮKassa + onboarding-tour
4. Если фокус = карты: первая задача — 2ГИС-парсер как новый Provider в стиле SearchProvider
