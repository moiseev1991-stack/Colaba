# План: Шаблоны коммерческих предложений (SEO)

**Дата:** март 2026  
**Область:** SEO, outreach, КП (коммерческие предложения)

---

## Ключевые решения

| Вопрос | Решение |
|--------|---------|
| Хранение шаблонов | В БД (бэкенд) |
| Выбор шаблона | При каждом запуске поиска |
| Плейсхолдеры | Да: `{{domain}}`, `{{issues}}`, `{{score}}` и др. |
| Подстановка значений | Бэкенд генерирует текст по шаблону |
| Редактирование перед отправкой | Да |
| Область | Пока только для SEO |
| Отправка | Пакетная (bulk) |

## Часть 1: Backend

### 1.1. Модель данных

**Новая таблица `outreach_templates`:**

| Поле | Тип | Описание |
|------|-----|----------|
| id | int | PK |
| organization_id | int | FK, мультитенантность |
| name | varchar | Название шаблона («SEO короткий», «Подробное КП») |
| subject | text | Шаблон темы письма (с плейсхолдерами) |
| body | text | Шаблон тела письма (с плейсхолдерами) |
| module | varchar | Пока только `"seo"` |
| is_default | bool | Шаблон по умолчанию (опционально) |
| created_at | datetime | — |
| updated_at | datetime | — |

### 1.2. API endpoints

| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET | `/api/v1/outreach/templates` | Список шаблонов (по org) |
| POST | `/api/v1/outreach/templates` | Создать шаблон |
| GET | `/api/v1/outreach/templates/{id}` | Получить шаблон |
| PATCH | `/api/v1/outreach/templates/{id}` | Обновить шаблон |
| DELETE | `/api/v1/outreach/templates/{id}` | Удалить шаблон |

### 1.3. Изменения в Search

- **POST `/api/v1/searches`** — принимать опционально `template_id?: number`.
- При генерации outreach в Celery использовать выбранный шаблон вместо hardcoded.
- Генерация: подставлять в шаблон плейсхолдеры.

### 1.4. Поддерживаемые плейсхолдеры

| Плейсхолдер | Описание | Источник |
|-------------|----------|----------|
| `{{domain}}` | Домен сайта | SearchResult |
| `{{issues}}` | Список SEO-проблем | extra_data / audit |
| `{{score}}` | SEO score (0–100) | seo_score |
| `{{company_name}}` | Название компании | title / extra_data |
| `{{title}}` | Title страницы | title |
| `{{url}}` | URL страницы | url |

## Часть 2: Frontend

### 2.1. Навигация

- Новый пункт в сайдбаре SEO: **Шаблоны КП** → `/app/seo/templates`.

### 2.2. Страница «Шаблоны КП»

**Путь:** `/app/seo/templates/page.tsx`

- Список шаблонов (карточки или таблица).
- Кнопка «Создать шаблон».
- Действия: Редактировать, Удалить, Сделать по умолчанию.

### 2.3. Создание/редактирование шаблона

**Пути:** `/app/seo/templates/new`, `/app/seo/templates/[id]/edit`

- Поля: Название, Тема письма, Тело письма.
- Справка по плейсхолдерам: `{{domain}}`, `{{issues}}`, `{{score}}`, `{{company_name}}`, `{{title}}`, `{{url}}`.
- Валидация: subject и body не пустые.

### 2.4. Выбор шаблона при запуске

**Страница:** `/app/seo/page.tsx`

- Селектор «Шаблон КП» (dropdown) рядом с провайдером, городом.
- Варианты: список шаблонов + «Без шаблона» (текущий генератор).
- Передавать `template_id` в `createSearch()`.

### 2.5. LeadsTable — bulk send + редактирование

- Выбор строк (чекбоксы уже есть).
- Кнопка «Отправить КП выбранным» (N шт).
- Модалка отправки:
  - Список получателей (домен, email).
  - Превью сгенерированного текста для каждого (можно редактировать).
  - Кнопка «Отправить всё».

### 2.6. API-сервис (frontend)

- `getOutreachTemplates()`, `createOutreachTemplate()`, `updateOutreachTemplate()`, `deleteOutreachTemplate()`.
- Расширить `createSearch()` — параметр `template_id?: number`.

## Часть 3: Этапы реализации

### MVP (первая итерация)

1. **Backend:**
   - Модель `OutreachTemplate`, миграция Alembic.
   - CRUD API для шаблонов.
   - Принимать `template_id` в создании поиска.
   - Модифицировать генерацию outreach в Celery — подстановка по шаблону.
2. **Frontend:**
   - Страница шаблонов + CRUD.
   - Селектор шаблона на странице SEO при запуске.
   - Bulk-отправка с модалкой (превью + редактирование).

### Второй этап (опционально)

- Экран «Кампании КП» — история рассылок, статистика.
- Экспорт/импорт шаблонов.
- Планирование отправки (отложенная рассылка).

## Реализовано на фронтенде (март 2026)

Следующие части уже есть в коде и будут работать после готовности бэкенда:

- **Страница шаблонов КП** (`/app/seo/templates`) — список, пустое состояние, кнопки создания/редактирования/удаления
- **Создание шаблона** (`/app/seo/templates/new`) — форма с названием, темой, телом и справкой по плейсхолдерам
- **Редактирование шаблона** (`/app/seo/templates/[id]/edit`) — та же форма для существующего шаблона
- **Пункт в сайдбаре** — «Шаблоны КП» в разделе SEO
- **Селектор шаблона на странице SEO** — выбор при запуске поиска, передача `template_id` в `config`
- **Bulk-отправка КП** — кнопка «Отправить КП (N)» при выборе строк, модалка с превью получателей и отправкой
- **API-клиент** — `getOutreachTemplates`, `createOutreachTemplate`, `updateOutreachTemplate`, `deleteOutreachTemplate`

При отсутствии бэкенда: список шаблонов пустой, создание/удаление показывают toast «API ещё не готово».
